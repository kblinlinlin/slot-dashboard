# 平台 Slot 游戏数据看板

这是一个内部使用的 Slot 游戏周数据看板 MVP。当前版本基于 `平台游戏数据看板.xlsx` 生成初始数据，并支持在页面内上传新的 Excel 表格进行刷新。

## 当前功能

- 游戏总览数据：参考 Excel `总览数据`，展示本周、上周和变化。
- 产商总览数据：参考 Excel `产商数据`，展示 AA、IGC、KK 的 Top 范围汇总。
- 游戏走势：选择一个或多个游戏、周期、指标后查看趋势。
- 产商走势分析：选择一个或多个产商、周期、Top 范围、指标后查看产商汇总趋势。
- 自定义周期：趋势页支持选择固定最近 N 周，也支持自定义开始周和结束周。
- 产商下注占比：产商走势页展示所选周期内 AA、IGC、KK 的总下注金额饼图。
- 自动总结：每次数据加载后，按固定结构生成本周对比上周的数据变化总结。
- 游戏名映射：优先使用 `游戏名称映射` sheet 的中文名；没有映射则显示英文名。

## 使用方式

当前本地服务地址：

```text
http://localhost:8765/
```

如果服务没有运行，可在项目目录执行：

```powershell
python -m http.server 8765
```

然后用浏览器访问上面的地址。

如果希望同一局域网内其他人访问，可运行：

```powershell
.\start-dashboard.ps1
```

然后让同事访问：

```text
http://你的电脑局域网IP:8765/
```

例如当前检测到的地址是 `http://192.168.2.124:8765/`。如果同事无法访问，通常需要在 Windows 防火墙中放行 8765 端口。

## 数据规则

- 数据以 Excel 表格中的脱敏数据为准。
- 上传完整看板时，页面会整体替换当前数据。
- 上传单周数据时，页面会按该周 `日期` 字段追加或覆盖同周期数据，并保存到当前浏览器本地存储。
- 单周数据至少需要包含周数据字段，例如 `日期`、`游戏ID`、`游戏名称`、`下注金额`、`游戏输赢`、`投注次数`、`下注金额排名变化`、`新增玩家`、`活跃玩家`。
- 除“新游上线监测”外，自动总结默认只分析本周 Top 50 游戏。
- 变化颜色规则：增长显示红色，降低显示绿色。
- 产商当前按游戏 ID 长度识别：
  - 7 位：IGC
  - 3 位及以下：AA
  - 6 位：KK

## 主要文件

- `index.html`：页面结构。
- `styles.css`：视觉和响应式样式。
- `app.js`：上传解析、表格渲染、趋势图、自动总结逻辑。
- `data/initial-data.js`：从当前 Excel 提取的初始数据。
- `tools/extract_workbook_data.py`：把 Excel 转成前端初始数据。
- `docs/slot-platform-dashboard-requirements.md`：需求文档。
